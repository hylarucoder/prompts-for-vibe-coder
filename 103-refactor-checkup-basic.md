# 代码重构体检 - 简单版

> 本提示词在 claude code + 最新的国产大模型中, 也适用于更高级别的模型， 注意, 只挑选了收益非常高的点。深入重构还请大家阅读代码整洁之道。

请基于《代码整洁之道》的原则，分析这个代码库的 [目录/文件]，找出以下问题：

## 1. 命名问题【有意义的命名】
- 变量/函数名是否清晰表达意图？
- 是否有 data1, temp, result 这类无意义命名？
- 同一概念是否用了多种命名方式？

## 2. 函数问题【函数应该短小】
- 是否有超过 100 行的函数？
- 一个函数是否只做一件事？（SRP 单一职责）
- 函数参数是否超过 3 个？

## 3. 重复问题【DRY 原则】
- 是否有多处几乎相同的代码逻辑？
- 是否可以提取成公共函数？

## 4. 过度设计【YAGNI 原则】
- 是否有不必要的兼容逻辑、try-catch、if-else？
- 是否有"以防万一"但从未使用的代码？

## 5. 魔法数字【避免硬编码】
- 是否有裸露的数字或字符串没有解释？
- 是否应该提取为常量？

请按以下格式输出：
- 问题类型
- 违反的原则
- 具体位置（文件:行号）
- 严重程度（高/中/低）
- 建议的修复方向
